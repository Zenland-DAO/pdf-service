# Zenland PDF Service

**Live:** [https://pdf.zen.land](https://pdf.zen.land)

The PDF Service generates **tamper-proof escrow agreement documents** for the Zenland V2 escrow platform. Each PDF serves as the official, verifiable record of an escrow contract.

## Why These PDFs Are Impossible to Fake or Modify

Every PDF generated by this service is **cryptographically signed and hash-locked to the blockchain**:

1. **Digital Signature** вҖ” Each PDF contains a cryptographic signature in its metadata, signed by Zenland's official key. The signature covers the entire document content, so any modification invalidates it.

2. **On-Chain Hash** вҖ” When an escrow is created, the `keccak256` hash of the PDF bytes is stored on-chain as `termsHash`. This creates an immutable link between the PDF and the smart contract.

3. **Verification** вҖ” Anyone with the PDF can verify:
   - The signature is valid and signed by Zenland
   - The PDF hash matches the `termsHash` stored on the blockchain
   - The document content matches the escrow parameters on-chain

**Result:** It's mathematically impossible to create a fake agreement that passes verification, or to modify an existing agreement without detection.

1. User enters escrow details and terms in the Zenland interface
2. Interface requests PDF generation from this service
3. Service generates a signed PDF and returns the raw bytes
4. Interface computes `termsHash = keccak256(pdfBytes)` client-side
5. User creates the escrow on-chain, which stores `termsHash`
6. The PDF exists only in the user's browser вҖ” **Zenland never stores your PDFs**

## Privacy by Design

- **No PDF storage** вҖ” PDFs are generated on-demand and sent directly to the user
- **No tracking** вҖ” No analytics, cookies, or user tracking
- **Client-side hashing** вҖ” The termsHash is computed in the user's browser

## Development

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Generate a preview PDF
npm run preview
```

## Environment Variables

```bash
# Required for production
PDF_SIGNING_PRIVATE_KEY=0x...   # Ethereum private key for signing PDFs
PDF_SIGNING_KEY_ID=production   # Key identifier embedded in metadata

# Optional
PORT=3002                        # Server port (default: 3002)
```

## API

**POST** `/api/generate` вҖ” Generate escrow agreement PDF

Returns PDF binary with headers:
- `Content-Type: application/pdf`
- `X-Generation-Time-Ms: <milliseconds>`

**GET** `/health` вҖ” Health check endpoint

## Architecture

This service is part of the Zenland V2 stack:

- **escrow-contracts/** вҖ” Solidity smart contracts
- **interface/** вҖ” Next.js web application
- **pdf-service/** вҖ” This service (PDF generation)
- **indexer/** вҖ” Blockchain event indexer
